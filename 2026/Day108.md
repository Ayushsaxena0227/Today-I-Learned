💀 The Root Cause: Why did this happen?
The startup likely made the #1 Rookie Mistake:
They used the API Key directly in their React (Frontend) Code.

❌ The Vulnerable Code (What they likely did):
JavaScript

// client/src/ChatComponent.js
import axios from 'axios';

const handleSend = async (message) => {
// 💀 FATAL ERROR: The key is bundled into the JS file sent to the browser!
const apiKey = "AIzaSyD-Gemini-Secret-Key-12345";

const response = await axios.post(
`https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent?key=${apiKey}`,
{ contents: [{ parts: [{ text: message }] }] }
);
};
Why is this bad?

Browser Visibility: Anyone can open Chrome DevTools -> Network Tab -> Filter "gemini". They will see the request URL with ?key=AIzaSy....
Source Code: Anyone can View Source or download your main.js bundle and search for "AIza".
Bot Scrapers: Hackers run bots that scan GitHub and live websites specifically looking for strings that start with AIza (Google), sk- (OpenAI), or AWS\_.
🛡️ The Solution: The Backend Proxy Pattern
The Interview Answer:
"To prevent this, I never call third-party APIs (like OpenAI, Gemini, or Payment Gateways) directly from the Frontend.
Instead, I use a Backend Proxy Pattern."

✅ Step 1: Store Key in Backend Environment Variables
Create a .env file on your server (Node.js). Add it to .gitignore so it never goes to GitHub.

env

# .env (Server Only)

GEMINI_API_KEY=AIzaSyD-Secret-Key-12345
✅ Step 2: Create a Proxy Route in Node.js
Your React app calls your server. Your server calls Gemini.

JavaScript

// server/routes/ai.js
import axios from 'axios';

router.post('/generate', async (req, res) => {
try {
const { prompt } = req.body;

    // 🔒 The key is used here, on the server. The user NEVER sees it.
    const response = await axios.post(
      `https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent?key=${process.env.GEMINI_API_KEY}`,
      { contents: [{ parts: [{ text: prompt }] }] }
    );

    res.json(response.data);

} catch (error) {
res.status(500).json({ error: "AI Service Failed" });
}
});
✅ Step 3: Frontend Calls Your Backend
JavaScript

// client/src/ChatComponent.js
const handleSend = async (message) => {
// ✅ Safe: We are calling our own backend. No keys exposed.
const response = await axios.post('/api/ai/generate', { prompt: message });
};
🛑 The Safety Net: Rate Limiting & Quotas
Even if you hide the key, a hacker could spam your /api/ai/generate endpoint, costing you money.

How to fix this (Recall our Rate Limiting lesson):

Rate Limiting (Middleware):
"I would apply express-rate-limit to the /api/ai/generate route. Each IP address can only send 10 requests per minute."

User Quotas (Business Logic):
"In EMS or Kontiggo, I would track usage in the database.

User.aiUsageCount: 50/100 used.
If User.aiUsageCount >= 100, reject the request at the backend level."
Cloud Hard Limits (The $82k Saver):
"In the Cloud Console (AWS/Google/OpenAI), I would set a Hard Billing Limit.

Set Monthly Budget: $50.
Action: If budget reached -> Disable API.
It's better to have the service stop working for a few hours than to go bankrupt."
